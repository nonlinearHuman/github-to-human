# 输出格式模板

每个项目输出时，严格按照以下格式：

## 完整格式

```markdown
### 🔥 {owner}/{repo}
🌟 {total_stars} stars | 📈 {today_stars} 今日增长

**说人话**：{一句话解释这东西是干什么的，普通人能 get 的价值点，要说结果不说过程}

**谁适合用**：
- {场景 1：具体说明能得到什么好处}
- {场景 2}
- {场景 3}

**为什么有意思**：{1-2 句话，说明这东西的独特价值或行业意义，可以用'相当于免费的XXX'或'以前需要XXX现在只要YYY'的句式}

🔗 {GitHub 链接}
```

## 示例

```
### 🔥 mattpocock/skills
🌟 85,072 stars | 📈 3,132 今日增长

**说人话**：把你的 AI 使用经验变成可复制的"技能包"，让团队里的每个人都能用上你探索出来的最佳实践。

**谁适合用**：
- 用 AI 编程但总是重复同样提示词的人（把好用的提示词存起来分享）
- 技术团队 leader（让全团队快速对齐 AI 使用方法）
- 独立开发者（积累自己的 AI 工作流，不依赖付费工具）

**为什么有意思**：相当于把你对 AI 的使用心得变成了一本开源 cookbook——以前你只能给同事口头讲，现在可以直接分享一串代码，他们也能用上你探索出来的好东西。

🔗 https://github.com/mattpocock/skills
```

```
### 🔥 supertone-inc/supertonic
🌟 6,081 stars | 📈 719 今日增长

**说人话**：让 iPhone 在不联网的情况下也能发出自然的人声，支持 10 种语言，完全免费开源。

**谁适合用**：
- 短视频创作者需要配音但不想花 TTS 订阅费
- 开发者想在 app 里加语音功能但不依赖云服务
- 记者/播客在旅行时快速生成配音

**为什么有意思**：这类工具以前只有云端收费版，现在在手机上本地跑，速度快还不限字数。说明 AI 能力正在从云端"落"到每个人的设备上。

🔗 https://github.com/supertone-inc/supertonic
```

## 翻译原则（注意事项）

1. **不说技术术语**：遇到 streaming、SSE、MCP、ONNX 等，直接跳过或用通俗说法替代
2. **说价值不说功能**：不说"这个工具支持 XX 功能"，说"这个工具能让你做 XX"
3. **说对比**：如果有商业竞品，说"相当于免费的 XXX"或"比 XXX 更好的是..."
4. **说规模**：提及 stars 数量，让普通人理解"有没有人用"
5. **一句话推荐要有钩子**：不是描述，是吸引。类似"这个工具让我每周省了 10 小时"
6. **谁适合用要具体**：不要"适合程序员"，要说"适合每周要写大量周报的职场人"
